Portal A management and overview application for telcos

Portal prototype

Introduction

Portal is a tool targeted at the telecommunication industry for management of subscriptions, data packages and users within a firm (B2B). First developed for iPinion ApS, it was purchased by IPVision A/S in November 2013. The tool offers overview of consumption and offers administration tools not only for client firms, but also for customer service representatives. My task was to build the front-end and deliver a complete system together with a dedicated team of back-end developers.

The process

The development process started with the creation of basic use case diagrams, that offered an overview of the system architecture for both front and back-end. Afterwards, it continued with corporate and competitors analysis which resulted in the first interface drafts. Wireframes were created next, followed by a design style that included interface elements from color schemes to icons or buttons. Finally, the first prototype front-end was created and approved. From then on it was merged with the back-end to continue the development process in a unitary manner. The extensibility and flexibility was tested heavily with many additions and changes being applied in the following months. However, the nature of the telco industry makes the application subject to constant change.

The dashboard

The homepage of the application (dashboard) offers overview of consumption for (by default) the current month of the entire company account. From the dashboard, certain periods can be selected as well as each user belonging to that company. The initial design envisioned a pie chart as the overview tool but in time it has been replaced with first a rectangle dashboard and finally by a horizontal one.

Current dashboard:

For company accounts, all iterations include a fast-switch option for users as well as an option to pick a custom date interval on the top left of the header. When pressed or touched, a list of users within the company is displayed so each can be chosen in order to view the consumption for the selected period. The application changes with each user, giving administrators control over all the accounts.

The users overlay not only displays all accounts created for the company, but offers an overview of subscriptions and internal extension numbers. User information can be edited and viewed here, from PIN and PUK codes, to application access (iOS application available for number forwarding), subscription history or account credentials.

Subscription / Addon change

Bellow the dashboard, controls for the current selected users are shown. From here, subscription can be modified and data packages can be added. The first iteration of the 'subscription change' was done by using a vertical layout and later changed to a horizontal one. (view changes)

Detailed consumption view

Besides the accounts administration, the user consumption is a very important component of the system. It offers detailed information about call, SMS, MMS and data usage. Each of the types are presented in three forms: line chart, bar chart and detailed view.

Mobile views

Although the design process did not follow a mobile first pattern, the application is capable of offering the same features for mobile devices which do not come equipped with large screens. The horizontal layout eased mobile front-end development, while preserving the functionality.

The challenge behind this project was to create a responsive front-end application that gives overview and control over a company's users. It had to offer a detailed representation of call, SMS, MMS and data consumption and allow users to quickly switch between subscriptions and addon plans.
